# 設計書

## アーキテクチャ概要

ファイル移動・差し替えのみ。コードの変更なし。

```
README.md（現在）→ docs/harness-guide.md に移動
README.md（新規）→ プレースホルダー形式に差し替え
```

## コンポーネント設計

### 1. docs/harness-guide.md（新規作成）

**責務**:
- platform-harness テンプレートの使い方ガイドを提供する
- テンプレート利用者がセットアップ手順を参照できる

**実装の要点**:
- 現在の README.md の内容をそのまま移動（タイトルを「platform-harness テンプレートガイド」に変更）
- `docs/` 配下に置くことで、テンプレート利用後もガイドとして参照可能

### 2. README.md（差し替え）

**責務**:
- テンプレート利用者が自分のプロジェクトの説明を書く場所を提供する
- テンプレートのセットアップガイドへの導線を提供する

**実装の要点**:
- プロジェクト名・説明をプレースホルダーにする
- 「このテンプレートについて」セクションに harness-guide.md へのリンクを設置
- 利用者がすぐに書き換えられるようにシンプルな構成にする
