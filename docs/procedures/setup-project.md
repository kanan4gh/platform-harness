# setup-project 手順（初回プロジェクトセットアップ）

プロジェクトの6つの永続ドキュメントを対話的に作成する手順書です。ハーネス中立の正であり、各ハーネスの呼び出し口(Claude Codeでは `/setup-project` コマンド)はこの手順書に対応します。

**実行タイミング**: テンプレートから新しいリポジトリを作成した直後。

## ステップ0: インプットの読み込み

1. `docs/ideas/` 内のマークダウンファイルを全て読む
2. 内容を理解し、PRD作成の参考にする(ファイルがなければ対話形式で要求を聞き取る)

## 承認ゲート(全ドキュメント共通)

各ドキュメントの作成後、選択肢を明確に提示して承認を確認し、ユーザーの回答を得てから次に進む:

```
質問: 「[ドキュメント名]を作成しました。内容を確認してください。どうしますか?」
選択肢:
  1. 承認して次へ (推奨)
  2. 修正を指示する → 修正点を聞き、反映後に再度この質問を行う
  3. このドキュメントをスキップ → 既定テンプレートのまま残し、次のステップへ
```

ハーネスが構造化された確認UI(選択式の質問ツール等)を提供する場合はそれを優先する(利用方法はハーネスアダプタが定義)。テキストで確認する場合も、上記の選択肢を明示して回答を待機する。

## ステップ1〜6: ドキュメントの作成

各ステップで対応する作成ガイド(Claude Codeではスキルとして提供)を読み込み、既存ドキュメントを踏まえて作成し、承認ゲートを実行する:

| ステップ | 作成ガイド | 生成ファイル | 備考 |
|---------|-----------|-------------|------|
| 1 | prd-writing | docs/product-requirements.md | `docs/ideas/` を元に、ユーザーストーリー・受け入れ条件・非機能要件・成功指標を具体化。**以降すべての土台になるためスキップ選択肢は提示しない**(承認/修正のみ) |
| 2 | functional-design | docs/functional-design.md | PRDを読んでから作成 |
| 3 | architecture-design | docs/architecture.md | 既存ドキュメントを読んでから作成 |
| 4 | repository-structure | docs/repository-structure.md | 同上 |
| 5 | development-guidelines | docs/development-guidelines.md | 同上 |
| 6 | glossary-creation | docs/glossary.md | 同上 |

## 完了条件

6つの永続ドキュメントが全て作成されていること。

完了時にユーザーへ報告する: 作成したドキュメントの一覧と、今後の使い方(ドキュメント編集は会話で依頼 / 機能追加は add-feature 手順[`docs/procedures/add-feature.md`] / レビューは review-docs 手順[`docs/procedures/review-docs.md`]。呼び出し表記は使用中ハーネスのエントリポイントに従う)。
